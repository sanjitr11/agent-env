## Architecture Overview

agent-env is a CLI tool with four distinct layers that compose cleanly. The entry point is `bin/agent-env.ts`, a thin command dispatcher that lazy-imports the right module for each subcommand (`init`, `sync`, `route`, `status`). All imports use dynamic `import()` so startup is fast regardless of which command runs.

The **config layer** (`src/config/`) is the core of the product. `schema.ts` defines the `StartupContext` type via Zod — this is the single source of truth that flows into everything else. `templates.ts` contains pure TypeScript template functions (no external engine) that take a `StartupContext` and return markdown strings for root CLAUDE.md, four agent CLAUDE.md files, and seven Claude Code slash commands. `generator.ts` orchestrates writing those strings to disk, with a `safeWrite` helper that respects `force` and `dryRun` flags and an invariant that agent CLAUDE.md files are written once and then owned by the agents.

The **state layer** (`src/state/`) is a SQLite store backed by Node 24's built-in `node:sqlite` (`DatabaseSync` — synchronous API). `db.ts` opens the connection, sets WAL mode, and runs schema migrations. `startup.ts` reads and writes the single-row `startup_context` table as a JSON blob. The database lives at `.agent-env/state.db` inside whatever project `agent-env init` was run in.

The **CLI layer** (`src/cli/`) contains the user-facing logic. `init.ts` runs the 7-question onboarding flow using `@clack/prompts`, then calls `makeStartupContext` → `saveStartupContext` → `generate`. `route.ts` implements weighted keyword scoring across four agent categories (coding / research / marketing / ops) with `@agent` prefix overrides and a `MIN_CONFIDENCE` fallback to ops. `sync.ts` loads context from SQLite and regenerates derived config files.

The **context layer** (`src/context/`) is a stub for Context-RAII integration. `session.ts` records agent sessions to SQLite (`agent_sessions` table). `scopes.ts` defines per-agent resource scope definitions. Neither is wired to the real Context-RAII library yet — the integration layer is clean but dormant.

## Code Conventions

- **ESM throughout** — `"type": "module"` in package.json, all imports use `.js` extensions on local files even though source is `.ts` (Node/tsx resolves at runtime)
- **Named exports only** — no default exports anywhere in `src/`
- **Errors thrown, not returned** — no `Result<T>` type; functions throw on invalid state
- **Zod at the boundary** — external input (onboarding answers, DB reads) is validated with Zod; internal function calls are trusted
- **Synchronous DB** — `DatabaseSync` from `node:sqlite` is used intentionally; CLI tools don't benefit from async DB calls and sync is simpler to reason about
- **No classes** — everything is plain functions and interfaces; the "agent" types in `src/agents/` are pure data (`Record<AgentType, string>` maps), not class instances
- **Template functions not template files** — CLAUDE.md content is generated by TypeScript functions in `templates.ts`, not Handlebars or similar
- **`picocolors` for terminal color** — not chalk; picocolors is zero-dep and tree-shakeable
- **Trailing newline on all generated files** — every template function ends its string with `\n`

## Known Gotchas

- `node:sqlite` prints `ExperimentalWarning` on every process start even in Node 24. Suppress with `NODE_NO_WARNINGS=1` or `--no-warnings=ExperimentalWarning`. Not suppressed yet — looks rough in demos.
- `bin/agent-env.ts` points `"bin"` in package.json to `./bin/agent-env.js` (compiled), but the project currently runs via `tsx` without a compile step. Running `npm start` will fail until `npm run build` is run first. For dev, always use `npx tsx bin/agent-env.ts`.
- `src/cli/sync.ts` is the `agent-env sync` command (regenerates config files from SQLite). It is NOT the `/sync` slash command (which analyzes the codebase). These share a name but are different things.
- The `agent_sessions` and `events` tables exist in SQLite but nothing reads from them yet — they're infrastructure for when Context-RAII is wired in.
